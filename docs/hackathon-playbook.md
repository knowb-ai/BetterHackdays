# Hackathon Playbook

## Goal

Turn a raw hackathon signup into a usable builder plan.

## Inputs

- event description
- rules or judging criteria
- deadline schedule
- team size limits
- allowed categories or tracks
- topic / niche preferences
- user profile or team profile

## Outputs

- recommended idea directions
- suggested team gaps
- prep checklist, detailed in [Prep Checklist](./prep-checklist.md)
- timeline with milestones
- focus areas for the first hours
- optional stretch guidance

## Experience shape

1. Collect the minimum required info.
2. Show a fast summary.
3. Offer optional deeper guidance.
4. Keep the plan short enough to act on immediately.
