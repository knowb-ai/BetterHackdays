# MVP Scope

## In scope

- Render-hosted MCP/API server as the default live service shape
- Hack Day session creation or wake-up flow
- active participants for connected but incomplete profiles
- matchable participants after profile completion
- matchmaking remains available
- event ingest stub
- concise hackathon prep guidance
- idea suggestions based on event + profile
- process suggestions with deadlines
- CLI or agent-harness flow for connect, survey, cards, likes, and matches
- team room creation after mutual match
- GitHub repo handoff plan with explicit permission

## Out of scope for now

- fully automated workspace provisioning
- heavy onboarding flows
- overly verbose planning screens
- advanced automation beyond the essentials
- provider-specific deployment assumptions as core product direction
- automatic GitHub repo creation without user approval
- IP-based identity or auth

## Success criteria

- users can get to a useful starting plan quickly
- users understand what to do next
- the system can support both solo builders and existing teams
- connected users have clear active, matchable, matched, and room states
- a mutual match can move toward a shared repo without manual confusion
